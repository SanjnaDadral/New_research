## 📋 DEPLOYMENT STEPS

### Step 1: Prepare Your Repository

```bash
# Ensure all changes are committed
git add .
git commit -m "Render deployment: Fix requirements, build script, and configuration"
git push origin main
```

### Step 2: Create Render Account & Project

1. Go to [render.com](https://render.com)
2. Sign up with GitHub
3. Create a new Blueprint from your repository
4. Select `New Blueprint Instance`

### Step 3: Configure in Render Dashboard

#### Database Setup
1. A PostgreSQL database will be automatically created
2. Render will set `DATABASE_URL` automatically ✓

#### Environment Variables (MUST SET MANUALLY)

Go to **Environment** tab in Render Dashboard and set:

```
SECRET_KEY=<auto-generated, keep it>
DEBUG=False
GROQ_API_KEY=<Get from https://console.groq.com>
EMAIL_HOST_USER=your-email@gmail.com
EMAIL_HOST_PASSWORD=<Gmail App Password, NOT your regular password>
DEFAULT_FROM_EMAIL=PaperAIzer <your-email@gmail.com>
```

#### How to Get Gmail App Password

1. Enable 2-Factor Authentication on Gmail
2. Go to [myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords)
3. Select "Mail" and "Windows Computer"
4. Generate and copy the 16-character password
5. Paste into `EMAIL_HOST_PASSWORD` in Render

#### How to Get Groq API Key

1. Go to [console.groq.com](https://console.groq.com)
2. Sign up with Google/GitHub
3. Create an API key
4. Copy and paste into `GROQ_API_KEY`

### Step 4: Deploy

1. Click **Deploy** in Render Dashboard
2. Wait for build to complete (~5-10 minutes)
3. Check **Logs** tab for any errors
4. Once deployed, get your app URL: `https://paper-analyzer-xxxx.onrender.com`

### Step 5: First-Time Setup

1. Visit your app URL
2. Create an admin account via Django admin panel
3. Configure any additional settings needed

## ⚠️ IMPORTANT NOTES

### Free Tier Limitations
- **Memory**: 512 MB (manage carefully)
- **CPU**: Shared instance
- **Spinning Down**: App goes to sleep after 15 mins of inactivity
- **Database**: 100 MB free (upgrade as needed)

### Recommendations
- ✅ Disable heavy ML features on free tier
- ✅ Limit PDF upload size to 45 MB
- ✅ Use lightweight document processing
- ✅ Keep session timeout reasonable
- ✅ Monitor database size regularly

### Performance Tips
1. Cache static files aggressively
2. Use PostgreSQL instead of SQLite
3. Monitor app logs for memory spikes
4. Consider upgrading if app gets popular

## 🐛 Common Issues & Solutions

### Issue: Build fails with "requirements.txt not found"
**Solution**: Ensure `requirements.txt` is in the root directory and committed to git

### Issue: Static files not loading
**Solution**: Run `python manage.py collectstatic --noinput` locally to test

### Issue: Email not sending
**Solution**: 
- Verify `EMAIL_HOST_PASSWORD` is an App Password (not regular Gmail password)
- Check Render logs for SMTP errors
- Whitelist Render's IP if using restricted SMTP

### Issue: Database connection error
**Solution**: 
- Ensure PostgreSQL database is created in Render
- Check `DATABASE_URL` environment variable
- Run migrations again: `python manage.py migrate`

### Issue: App spins down after 15 minutes
**Solution**: This is normal on free tier. Use paid tier to keep app running 24/7
